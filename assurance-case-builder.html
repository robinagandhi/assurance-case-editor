Assurance Case Confidence Maps Builder — Help

Getting Started

  • Double-click the canvas to add a Claim node at that position.
  • Click a node type in the left palette to add it at the canvas center, then fill in its text — or drag a palette button onto the canvas to drop it at an exact spot.
  • Double-click any node to edit its label and text in a popup, or edit directly in the Properties sidebar on the right.
  • Right-click a node for a quick menu: Duplicate, Copy, Select Subtree, change type, or Delete. Right-click empty canvas to add a Claim there or paste the clipboard.

Connecting Nodes

  • Hover a node to reveal small ○ handles at its four edges. Drag from a handle to another node to draw an arrow.
  • Alternatively, activate → Connect mode in the left panel, then click the source node, then click the target node.
  • To remove a connection, activate ✕ Delete mode and click the arrow line, or right-click the line and choose Delete Connection.
  • Hover or select a Claim, Defeater, or Inference Rule to reveal a small + button to its right — click it to add a new connected sub-claim below it and open the editor immediately. (Change its type from the editor's Type dropdown if you meant Evidence or a defeater instead.)

Selecting Multiple Nodes

  • Shift-click a node to add or remove it from the current selection.
  • Shift-drag on empty canvas to draw a marquee box — every node it touches is added to the selection. (Plain drag on empty canvas still pans, as usual.)
  • Drag any node that's part of a multi-selection to move the whole group together.
  • Arrow keys nudge the current selection by a few pixels; hold Shift for a bigger step.
  • Delete/Backspace removes every selected node (and their connections) at once.

Duplicate, Copy & Paste

  • Ctrl D / ⌘ D — duplicates the current selection (nodes + the connections between them) just below/right of the original.
  • Ctrl C / ⌘ C then Ctrl V / ⌘ V — copies the selection to an in-memory clipboard; paste repeatedly to stamp down more copies, each offset a little further. Right-click empty canvas and choose Paste to drop the clipboard at that exact spot instead.
  • Right-click a node and choose Select Subtree to select it and everything connected below it — handy before copying or duplicating a whole branch.

Navigating the Canvas

  • Drag empty canvas to pan in any direction. On a touchscreen, drag with one finger to pan/select and pinch with two fingers to zoom.
  • Scroll to zoom in and out (zooms toward the cursor).
  • Fit button (bottom bar) — fits all visible nodes into view (collapsed branches are ignored).
  • Auto Layout — arranges nodes as a top-down tree: Claims → Sub-claims → Defeaters → Evidence. Context and Justification nodes are placed to the right of their parent.
  • Find in diagram (top-right of the toolbar) — type to highlight every node whose label or text matches; press Enter to jump to the next match (Shift Enter for the previous), Escape to clear.

Outline View & Collapsing Branches

The sidebar's Outline tab (next to Checklist) lists every node as a nested, clickable tree — useful for jumping around a large case without panning. Clicking a row (or a node in the Checklist) automatically re-expands any collapsed ancestor so the target is visible.

On the canvas, any node with outgoing connections shows a small −/+ toggle below it (in the same spot the undeveloped-branch diamond would otherwise occupy). Click it to collapse or expand everything below that node — handy for decluttering a big diagram while you work on one branch. Collapse state is just a view preference; it isn't saved to the .gsn.json file or the exported SVG.

Node Types

TypeShapePurpose
C — ClaimRectangleA proposition about a critical system property. Written as noun-phrase + verb-phrase. Example: "The system minimizes information disclosure."
E — EvidenceCircleTangible, observable support for a claim. Written as a noun-phrase only. Example: "Test results from penetration tools."
CT — ContextDog-ear boxScope or definition that helps interpret a claim. Example: "Supply chain process as defined in NIST SP 800-160."
J — JustificationEllipseRationale for why a claim was introduced. Example: "Intelligence reports of malicious code in nuclear energy software."
R — RebuttingLight yellow boxUnless… doubt that challenges a claim. Example: "Unless Tweety is a penguin."
U — UnderminingLight blue boxUnless… doubt that challenges evidence. Example: "Unless the DNA test was contaminated."
UC — UndercuttingLight green boxUnless… doubt that challenges an inference rule. Example: "Unless Tweety is a juvenile bird."
IR — InferenceParallelogramMakes an implicit inference rule explicit so it can be challenged. Example: "If Tweety is a bird then it can fly."

Branch Indicators (automatic)

  • ◆ Black diamond — appears automatically below any Claim or Defeater with no outgoing connections, marking the branch as undeveloped. Disappears when you connect something below it.
  • ⬤ Grey circle — appears automatically below any Evidence node with no outgoing connections, marking the branch as complete. Disappears when you connect something below it.

Writing Good Claims

Claims must be about critical properties, not methods or actions. They must be falsifiable (true or false upon investigation).
Good ✓Bad ✗
"The login page has no exploitable SQL injection weakness""Perform SQL injection testing on the login page" (action, not a claim)
"The system minimizes information disclosure during communication""The system uses AES encryption" (trivially observable, no argument needed)
"Canvas attack surface is acceptably minimized""XSS results for Canvas" (no verb phrase)

Baconian Confidence

Displayed in the right sidebar as addressed | total (e.g., 2|3).

  • Every Rebutting, Undermining, and Undercutting defeater counts as one doubt.
  • A doubt is addressed when the defeater node has at least one outgoing connection (a sub-claim or evidence responding to it).
  • Green ≥ 80% · Orange 40–79% · Red < 40%

Case Checklist & Claim Tips

Below the confidence meter, the Case Checklist scans your diagram for common structural issues: disconnected nodes, undeveloped branches, missing descriptions, top-level claims with no Context, Claims that read like an action/method instead of a property, and Evidence that reads like a full sentence instead of a noun-phrase. Click any item to jump to that node.

When you add or edit a node — in the popup editor or the right-hand Properties panel — you'll see a short writing tip for that node type with a good example, plus a live warning if your text looks off.

These are heuristic nudges based on simple pattern-matching, not hard rules or NLP — they will miss real issues and occasionally flag fine text. Use your judgment as the case author; nothing is blocked from saving.

Review Mode (Reviewer Notes)

Toggle Review Mode in the toolbar to reveal a Reviewer Note field — separate from the student's own claim/evidence text — in both the popup editor and the Properties sidebar. This is meant for instructor or peer-review feedback on a specific node without editing the node's actual content.

Any node with a note shows a small amber ! badge in its top-right corner at all times, even with Review Mode off, so reviewer comments are easy to spot while browsing a case.

Presenter Mode

Click ▶ Present (bottom floating toolbar) to step through the case top-down, one node at a time, in the same order as the Outline view. The current node is highlighted with a purple ring and everything else dims; a caption bar at the top shows its label and text.

  • → / Space — next node  ·  ← — previous node  ·  Escape — exit
  • Editing, dragging, and right-click are disabled while presenting so nothing shifts mid-walkthrough; manual zoom/scroll still works.
  • Exiting restores whatever pan/zoom you had before you started presenting.

Save & Load

  • Save to Disc (Ctrl/⌘ S) — downloads your diagram as a .gsn.json file to your computer. Use this to back up or share your work.
  • Open File — opens a previously saved .gsn.json file and restores your diagram.
  • Save SVG — exports the complete diagram as a scalable vector graphic (.svg) suitable for reports, slides, or printing. Handles and selection are not included.
  • Load Sample — loads the Tweety Can Fly example from the course lecture to explore.
Your work is also auto-saved in the browser as a crash recovery backup. If you reopen the tool and a backup is found, you'll be asked whether to restore it. This is separate from the disc file — always use Save to Disc to keep a permanent copy.
